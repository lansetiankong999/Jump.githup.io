在一个系统运行期间，某个类只需要一个类实例就可以了，这时我们就可以使用单例模式。

1.定义：保证一个类仅有一个实例，并提供一个访问它的全局访问点。

2.解决方案：(全局共享点getInstance)

懒汉式：私有化构造方法，只有在类调用时才去创建实例

饿汉式：私有化构造方法，类加载器加载时就进行创建实例(能够解决多线程问题)

懒汉式体现了lazyLoad(延迟加载)，缓存的思想，lazyLoad为了尽可能的节约资源

Java中最常见的实现缓存方式就是Map(如果存在该资源就直接使用，不存在就创建再使用，并且下次还能继续使用)

3.利用缓存来实现单例模式

4.单例模式的优缺点

时间和空间，线程安全(懒，饿汉分别有不同的优缺点)

如何实现延迟加载和线程安全同时存在呢—->Lazy initialization holder class
`
public class Singleton{
private static class SingletonHolder{
private static Singleton instance = new Singleton();
}
private Singleton(){}
public static Singleton getInstance(){
return SingletonHolder.instance;
}
}
